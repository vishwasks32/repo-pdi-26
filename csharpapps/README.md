# C# Applications Collection

A collection of C# applications demonstrating various programming concepts, frameworks, and architectures.

## Projects Overview

### Console Applications

#### [HelloWorld](./HelloWorld/)
A basic console application demonstrating namespaces and object-oriented programming fundamentals.

#### [Grade Calculator](./grade_calculator/)
Calculates student GPA based on courses, grades, and credit hours.

#### [IPv4 Validator](./Ipv4Validator/)
Validates IPv4 addresses according to standard formatting rules.

#### [Banking App](./BankingApp/)
Demonstrates inheritance and polymorphism with bank account management.

#### [Username Extractor](./usernameExtracter/)
Extracts username from email addresses with basic validation.

#### [Vault Manager](./VaultManager/)
Secure password manager with encryption and persistent storage.

### Web Applications

#### [ASP.NET Core MVC App](./aspcoreapp1/)
Basic MVC web application with Razor views and standard ASP.NET Core features.

#### [Kirana App](./kiranaapp/)
RESTful API for product management in a grocery store with Swagger documentation.

#### [Random User API](./randomuser/)
Web API integrating with external services for user data and weather forecasts.

#### [Static Website](./StaticWebsite/)
Simple static website built with ASP.NET Core Razor Pages.

#### [Kirana App v1](./KiranaAppv1/)
Full-featured layered architecture application with authentication, database integration, and comprehensive API.

## Technologies Used

- **Languages**: C#
- **Frameworks**: .NET 8.0, .NET 9.0, .NET 10.0, ASP.NET Core
- **Databases**: MySQL, In-memory storage
- **Authentication**: JWT, ASP.NET Core Identity
- **Documentation**: Swagger/OpenAPI
- **Validation**: FluentValidation
- **Mapping**: AutoMapper
- **Testing**: xUnit

## Prerequisites

- .NET SDK (versions 8.0, 9.0, or 10.0 depending on project)
- MySQL Server (for database-dependent projects)

Each project contains its own README with specific setup and running instructions. Navigate to the project directory and follow the individual documentation.
